9.1.2 dbt — ADD these calibrated answers:
  ✓ Materialization trade-off
    view vs incremental vs table
    view chaining = performance degradation
    storage $23/TB vs compute credits
    
  ✓ Iceberg feature boundary
    staging = handoff point
    filter pushdown through view to Iceberg
    int/mart = Snowflake native behavior
    on_schema_change bridges the gap

  ✓ Silver layer design decisions
    quality filters in intermediate not staging
    staging silent drop = no audit trail
    quarantine = live DQ window
    enrichment joins in silver not marts
    surrogate key = immutable columns only

  ✓ Surrogate key edge case
    mutable columns = hash collision risk
    pickup_datetime correction = new trip_id
    immutable ingestion_ts anchors quarantine key

  ✓ ETL → dbt parallel
    Ab Initio dedup → intermediate SQL
    UPSERT → dbt MERGE via unique_key
    SCD Type 2 flag → dbt snapshots
    logic in tool → logic in version-controlled SQL
